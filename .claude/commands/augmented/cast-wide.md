# Cast Wide - 대안 탐색 강제

출처: [Augmented Coding Patterns](https://lexler.github.io/augmented-coding-patterns/patterns/cast-wide/)

## 문제

첫 번째 해결책은 제한된 지식에서 나온다. AI도 가장 익숙한 패턴을 먼저 제시하므로, 수용하면 더 나은 대안을 놓칠 수 있다 (Answer Injection 안티패턴).

## 프로세스

### 1단계: 초기 접근법 논의 후 블라인드 스팟 탐색

다음 질문을 순차적으로 적용:

1. "우리가 고려하지 않은 대안적 해결책이 있는가?"
2. "이 문제에 대해 우리가 생각하지 못하고 있는 것은?"
3. "더 단순하게 할 수 있는가?" → "더 더 단순하게는?"
4. "완전히 다른 카테고리의 해결책은?"

### 2단계: 컨텍스트 검증

각 대안을 현재 컨텍스트에 대입:
- 이 솔루션이 동작해야 하는 환경은?
- 실제 제약 조건(인원, 시간, 인프라)은?
- 엔터프라이즈 수준이 정말 필요한가, 아니면 단일 머신이면 충분한가?

### 3단계: 결정

- 최소 3개 이상의 접근법을 비교 테이블로 정리
- 각 접근법의 복잡도, 유지보수 비용, 제약 적합성을 평가
- 가장 단순하면서 제약에 맞는 접근법 선택

## 예시

이메일 처리가 필요한 상황:
- 1차 제안: Microsoft Graph API + OAuth2 + Azure AD (과도)
- 2차 제안: IMAP + basic auth (여전히 복잡)
- 3차 발견: 파일 모니터링 — Outlook이 이메일을 폴더에 저장하도록 설정 후 파일 처리

→ 컨텍스트(단일 PC)에 맞는 가장 단순한 해결책 발견

## 관련 패턴

- **해결**: Answer Injection 안티패턴
- **연관**: `superpowers:brainstorming` (brainstorming 중 대안 탐색 강화에 활용)
