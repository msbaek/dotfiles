# Softest Prototype - 마크다운 프로토타이핑

출처: [Augmented Coding Patterns](https://lexler.github.io/augmented-coding-patterns/patterns/softest-prototype/)

## 문제

코드는 너무 일찍 제약한다. 뭘 만들어야 하는지 만들어봐야 알지만, 만드는 데 시간이 걸리고 잘못된 결정은 바꾸기 비싸다.

## 핵심 원칙

> 코드 대신 마크다운 지시 파일로 AI를 유도하면, 컴파일/리팩토링 없이 즉시 방향을 바꿀 수 있다.

## 프로세스

### 1단계: 마크다운 지시 파일 작성

```markdown
# [기능명] 프로토타입 지시

## 목표
[달성하려는 것]

## 규칙
- [동작 규칙 1]
- [동작 규칙 2]

## 입력 예시
[사용자가 줄 수 있는 입력]

## 기대 출력
[AI가 생성해야 하는 출력]
```

### 2단계: AI에게 지시 파일 따르게 하기

```
이 지시 파일을 읽고 그대로 따라줘.
[지시 파일 내용 또는 경로]
```

### 3단계: 사용하면서 발견

- 실제 사용 중 누락된 요구사항 발견
- 마크다운 파일 즉시 수정 → 방향 전환
- 컴파일 지연, 리팩토링 오버헤드 없음

### 4단계: 코드로 전환 결정

충분히 탐색한 후:
- 요구사항이 안정화되면 실제 코드로 구현
- 프로토타입에서 발견한 규칙이 코드 설계의 기초가 됨

## 적용 시나리오

| 상황 | 방법 |
|------|------|
| 새 기능 탐색 | 마크다운으로 규칙 정의 → AI 테스트 → 규칙 정제 |
| 워크플로우 설계 | 입출력 형식만 마크다운으로 정의 → AI가 실행 |
| 게임 프로토타입 | 게임 규칙을 마크다운으로 → AI와 플레이하며 규칙 발견 |
