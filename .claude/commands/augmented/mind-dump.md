# Mind Dump - 비구조화 사고 캡처

출처: [Augmented Coding Patterns](https://lexler.github.io/augmented-coding-patterns/patterns/mind-dump/)

## 문제

정리된 프롬프트를 작성하려면 이미 생각이 정리되어 있어야 한다. 하지만 초기 아이디어 단계에서는 생각이 정리되지 않은 경우가 대부분이다.

## 핵심 원칙

> 정리하지 말고, 구조화하지 말고, 그냥 쏟아내라. AI가 신호에서 노이즈를 분리한다.

## 프로세스

### 1단계: 쏟아내기

아무 구조 없이 떠오르는 대로 입력:
- 음성 입력(Whispr Flow, 받아쓰기) 활용 가능
- 문법, 구조, 순서 무시
- "~인 것 같은데", "아 그리고", "잠깐 이것도" 모두 허용

### 2단계: AI에게 구조화 요청

```
이 내용에서 핵심 요구사항, 제약 조건, 우려 사항을 추출해줘.
```

### 3단계: 후속 대화로 정제

```
내용에 대해 질문해줘. (Ask me questions)
```

→ 독백(monologue)이 구조화된 대화(dialogue)로 변환

## 활용 시나리오

| 상황 | 방법 |
|------|------|
| 프로젝트 컨텍스트 로딩 | 산책하며 프로젝트 상세 음성 입력 → AI가 요구사항 추출 |
| 디버깅 | 문제 상황을 말로 풀어내기 → AI가 핵심 이슈 식별 |
| 하루 마무리 (저에너지) | 음성 입력으로 타이핑 피로 우회 |
| 요구사항 수집 | 이해관계자 대화 녹음 → AI가 유저 스토리 도출 |
| 아침 계획 | 우선순위와 블로커 말하기 → AI가 실행 계획 구조화 |
