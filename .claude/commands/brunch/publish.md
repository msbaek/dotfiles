# 브런치 발행 준비 (Phase 5: Publish)

제목 후보와 발행 체크리스트를 제공합니다.

## 사용법
```
/brunch:publish
```

---

## 수행할 작업

### 1. 제목 후보 제안

기존 브런치 제목 패턴 분석:
- **짧고 직접적**: "테크니컬 코치", "2023 회고"
- **질문형**: "숙련된 개발자의 역할은?"
- **대비/역설**: "TDD를 배우기 어려운 이유", "몰입의 함정"
- **부제목 활용**: "주제 | 부연 설명"

```markdown
## 제목 후보

### 직접적 스타일
1. **[제목1]**
2. **[제목2]**

### 질문형 스타일
3. **[제목3]?**

### 대비/역설 스타일
4. **[제목4]**

### 부제목 포함
5. **[제목5]** | [부제목]
```

### 2. 발행 체크리스트

```markdown
## 발행 체크리스트

### 제목
- [ ] 길이 적절 (30자 내외 권장)
- [ ] 핵심 키워드 포함
- [ ] 호기심 유발

### 본문
- [ ] 오탈자 확인
- [ ] 링크 동작 확인
- [ ] 이미지 삽입 위치 적절
- [ ] 굵은 글씨 강조점 확인

### 메타 정보
- [ ] 브런치 매거진 선택 (해당시)
- [ ] 키워드/태그 설정
- [ ] 공개 범위 설정

### 최종 확인
- [ ] 모바일에서 읽기 편한지
- [ ] 첫 문단이 훅으로서 충분한지
```

### 3. Obsidian 저장 (선택)

```markdown
## Obsidian 저장 옵션

브런치 발행 전/후에 Obsidian vault에 저장하시겠어요?

저장 위치 제안:
- `~/DocumentsLocal/msbaek_vault/003-RESOURCES/BRUNCH/[제목].md`

저장하면 다음이 추가됩니다:
- 브런치 URL (발행 후)
- 작성일
- 관련 태그
```
