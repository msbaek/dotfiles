# 브런치 글 섹션 작성 가이드 (Phase 3: Flesh)

특정 섹션 작성을 위한 체크리스트와 유도 질문을 제공합니다.

## 사용법
```
/brunch:flesh [섹션번호]
```

예시: `/brunch:flesh 2`

---

## 수행할 작업

### 1. 해당 섹션 정보 확인
$ARGUMENTS 번째 섹션의 구조 확인

### 2. 체크리스트 + 유도 질문 제공

```markdown
## 섹션 $ARGUMENTS 작성 가이드

### 이 섹션의 역할
[이전 skeleton에서 정의한 역할]

### 체크리스트
- [ ] 개인 경험/에피소드 포함
- [ ] 구체적 예시/숫자 (있으면)
- [ ] 질문형 문장 1개 이상
- [ ] 핵심 메시지 굵은 글씨 강조
- [ ] 레퍼런스 링크 (필요시)

### 유도 질문
1. "이 상황에서 어떤 감정을 느꼈나요?"
2. "왜 이것이 중요하다고 생각하시나요?"
3. "독자에게 전달하고 싶은 핵심 메시지는?"
4. "구체적인 숫자나 사례가 있나요?"

### 참고할 vault 문서
- [문서1.md]: "인용할 수 있는 문장"
- [문서2.md]: 관련 개념 설명

### 작성 팁 (백명석님 스타일)
- 1인칭 사용 ("나는", "제 경험상")
- 구어체로 자연스럽게
- "왜 그럴까?", "어떻게 해야 할까?" 같은 질문
- 구체적인 상황 묘사 (누가, 언제, 어디서)

### 초안을 작성해주시면 피드백 드리겠습니다.
직접 글을 쓰지 않고, 작성하신 초안에 대한 피드백만 제공합니다.
```

---

## 핵심 원칙
- **절대 직접 글을 쓰지 않음**
- 질문으로 유도
- 사용자가 작성한 후 피드백 제공
- 개인 경험/감정 부분은 사용자만 작성 가능
