# 브런치 글 구조 제안 (Phase 2: Skeleton)

이전 단계(start)의 결과를 바탕으로 글 구조를 제안합니다.

## 사용법
```
/brunch:skeleton
```

---

## 수행할 작업

### 1. 이전 단계 결과 확인
- 추출된 키워드
- vault에서 찾은 관련 문서
- 사용자의 핵심 메시지

### 2. 3-5개 섹션 구조 제안

다음 형식으로 제안:

```markdown
## 글 구조 제안: "[주제]"

### 마인드맵 (텍스트)
```
                    [핵심 주제]
                         │
         ┌───────────────┼───────────────┐
         ▼               ▼               ▼
    [1. 훅/문제제기]  [2. 본론]     [3. 결론]
         │               │               │
    - 개인 경험      ┌───┴───┐      - 실천 방안
    - 질문 던지기    ▼       ▼      - 여운
               [사례1] [사례2]
```

### 섹션별 가이드

#### 1. 훅 (Hook) - 문제 제기
- **역할**: 독자의 관심 끌기
- **필요한 것**: 개인 경험 또는 에피소드
- **참고 vault 문서**: [문서명]
- **질문**: 어떤 경험으로 시작하시겠어요?

#### 2. 본론 - 사례/논거
- **역할**: 주장 뒷받침
- **필요한 것**: 2-3개 사례 또는 레퍼런스
- **참고 vault 문서**: [문서명1], [문서명2]
- **질문**: 어떤 사례를 강조하고 싶으신가요?

#### 3. 결론
- **역할**: 핵심 메시지 전달
- **필요한 것**: 명확한 takeaway 1-2개
- **질문**: 독자가 얻어갔으면 하는 것은?

### 이 구조로 진행할까요?
수정하고 싶은 부분이 있으면 말씀해주세요.
구조가 확정되면 `/brunch:flesh 1` 로 첫 번째 섹션부터 시작합니다.
```

## 백명석님 브런치 글 구조 패턴

기존 글 분석 결과:
1. **시간순 전개**: 경험 → 과정 → 교훈
2. **에피소드 중심**: 구체적 사례 2-3개
3. **핵심 메시지 강조**: 굵은 글씨
4. **부록/추가 에피소드**: 관련 이야기 덧붙임
