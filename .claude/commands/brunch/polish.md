# 브런치 글 다듬기 (Phase 4: Polish)

작성된 글의 스타일을 체크하고 개선점을 제안합니다.

## 사용법
```
/brunch:polish
```

---

## 수행할 작업

### 1. 백명석님 스타일 체크리스트 확인

```markdown
## 스타일 체크 결과

### 어조 체크
- [ ] 1인칭 사용 ("나는", "제 경험상", "내 생각")
- [ ] 구어체 자연스러움 (딱딱하지 않음)
- [ ] 질문형 문장 포함 ("왜 그럴까?", "어떻게 해야 할까?")

### 구조 체크
- [ ] 시간순/인과관계 전개 (경험 → 과정 → 교훈)
- [ ] 에피소드/사례 최소 2개
- [ ] 핵심 메시지 굵은 글씨 강조
- [ ] 전문 지식 참고자료 인용 (책, 영상, 블로그 등)

### 마무리 체크
- [ ] 실천 가능한 제안 또는 인사이트
- [ ] 열린 결말 또는 여운 (부록/추가 에피소드 등)

### 금지 표현 체크
- [ ] "delve", "comprehensive", "leverage" 등 AI 표현 없음
- [ ] 과도한 형용사/수식어 없음
- [ ] 문장이 자연스럽고 백명석님이 직접 쓴 것 같음
```

### 2. 구체적 피드백 제공

```markdown
### 잘된 부분
- ✅ [구체적으로 잘된 부분]
- ✅ [구체적으로 잘된 부분]

### 개선 제안

#### 1. [위치/섹션]
**현재**: "[현재 문장]"
**제안**: 기존 스타일처럼 [개선 방향] 추가하면 어떨까요?
- 기존 글 예시: "[유사한 패턴의 기존 문장]"

#### 2. [위치/섹션]
**현재**: "[현재 문장]"
**제안**: 핵심 메시지를 굵은 글씨로 강조하면 좋을 것 같아요.
- 제안: **"강조할 문장"**

### 수정 여부는 직접 결정해주세요.
위 제안을 반영하실지는 전적으로 사용자님의 선택입니다.
```

---

## 핵심 원칙
- 제안만 하고 직접 수정하지 않음
- 스타일 강제 변경 금지
- 사용자 원문 존중
