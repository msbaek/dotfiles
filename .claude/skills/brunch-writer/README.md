# brunch-writer

> 브런치 블로그 글 작성을 보조하는 Claude 스킬. 직접 글을 쓰는 대신 자료 수집, 구조 제안, 스타일 체크로 사용자의 글쓰기를 돕습니다.

## 만든 배경

브런치 글 작성 시 Claude 초안과 본인 수정본의 차이를 분석하면서 생성되었습니다. 핵심 철학은 **"Claude는 정원사, 사용자는 나무"** — 사용자의 경험과 생각이라는 씨앗을 받아 vault-intelligence로 자료를 수집하고, 구조화와 다듬기를 제안하여 글이 자라도록 돕는 역할입니다.

## 사용법

### 호출 방법

초안 파일(.md)을 @멘션으로 제공하면서 "브런치", "글 작성", "블로그 글" 등의 키워드를 언급하면 자동으로 활성화됩니다.

### 예시

```
@초안.md 이 내용으로 브런치 글을 작성하려고 하는데 도움 줄 수 있어?
```

```
@draft-tdd.md 블로그 글로 다듬어주세요
```

## 주요 기능

### 5단계 워크플로우

1. **Seed (씨앗 심기)**: 초안 분석 → 핵심 키워드 추출 → vault-intelligence로 관련 문서 검색 → 참고자료 목록 제공
2. **Skeleton (뼈대 세우기)**: 3-5개 섹션 구조 제안 → 텍스트 마인드맵 시각화 → 각 섹션 가이드
3. **Flesh (살 붙이기)**: 섹션별 체크리스트 → 유도 질문으로 작성 독려 → vault 인용문 제안
4. **Polish (다듬기)**: 백명석님 스타일 체크리스트 확인 → 어색한 표현 지적 → 굵은 글씨 강조점 제안
5. **Publish (발행 준비)**: 제목 후보 3-5개 제안 → 브런치 포맷 확인 → 태그/키워드 제안

### 핵심 원칙

**DO (해야 할 것)**
- 질문으로 유도: "이 부분에 어떤 경험을 넣으시겠어요?"
- 선택지 제공: "A 또는 B 중 어떤 방향이 맞나요?"
- vault 자료로 영감 제공: "이런 자료가 있는데 활용하시겠어요?"
- 체크리스트로 빈 곳 알려주기: "이 부분이 빠진 것 같아요"

**DON'T (하지 말아야 할 것)**
- 문장 직접 작성 금지: 특히 개인 경험, 감정 부분
- 스타일 강제 변경 금지: 어색해도 사용자 원문 존중
- 구조 강요 금지: 제안은 하되 결정은 사용자
- AI 특유의 표현 사용 금지: "delve", "comprehensive", "leverage" 등
- 추상적/모호한 결과 표현 금지: "아무것도 못 배운다" 대신 "일은 시작도 못 할 것이다"처럼 구체적 결과로
- 실명 노출 금지: 공인이 아닌 한 익명화
- 절대적 표현 금지: "거기서 끝"보다 "점점 좁아질 가능성이 높다"처럼 뉘앙스 있게

### 백명석님 스타일 체크 포인트

- 1인칭 사용 ("나는", "제 경험상")
- 구어체 자연스러움
- 질문형 문장 포함
- 시간순/인과관계 전개 (경험 → 과정 → 교훈)
- 에피소드/사례 최소 2개
- 핵심 메시지 굵은 글씨 강조
- 자기만의 용어 브랜딩 (예: "내가 '조각 지식 습득하기'라고 부르는 방법")
- 문화적 레퍼런스 추가 (드라마, 영화, 한국 문화 인용)
- 인용구에 출처 귀속 명시
- 추상적 결과 → 구체적 결과
- 절대적 표현 → 뉘앙스
- 기술 용어에 영어 병기 (예: "옆(horizontal)으로")
- 공인이 아닌 인물 익명화

## 의존성

| 도구/서비스 | 용도 |
|------------|------|
| vault-intelligence | 관련 문서 검색, MOC 생성, 쿼리 확장 |
| Obsidian vault | 참고 자료 저장소 ($VAULT_ROOT) |

### vault-intelligence 명령어

```bash
cd ~/git/vault-intelligence

# 키워드 검색 (hybrid + rerank)
python -m src search --query "검색어" --search-method hybrid --top-k 20 --rerank

# 쿼리 확장 (동의어 + HyDE)
python -m src search --query "검색어" --expand

# MOC 생성
python -m src generate-moc --topic "주제" --top-k 30

# 관련 문서 찾기
python -m src related --file "파일명.md" --top-k 10
```
