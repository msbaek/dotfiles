# Write-In-My-Voice — 백명석 스타일 글쓰기 어시스턴트

<!-- Execution: interactive (main context). frontmatter `model:` 필드는 main context 호출 시 무시되므로 제거. multi-turn 대화 유지를 위해 sub-agent 위임 안 함. -->

**Voice Profile 경로:** `~/.claude/skills/humanize-korean/references/voice-profile-msbaek.md`

이 스킬은 **2단계 파이프라인**으로 동작한다. 구조를 먼저 확정하고 본문을 작성해
사용자가 방향을 조정할 기회를 갖는다.

---

## Phase 1: 구조 설계

**실행 모드:** 단일 (메인 세션)

1. `voice-profile-msbaek.md`를 읽어 주제 영역·서사 구조·어조 특징을 파악한다.
2. 사용자 입력(주제/키워드)을 분석한다.
3. **제목 후보 3개를 제시한다:**
   - 선언형 (예: "TDD는 설계 행위다")
   - 질문형 (예: "왜 TDD는 배우기 어려울까?")
   - 경험 서사형 (예: "처음 TDD를 만났을 때")
4. **섹션 구조 초안을 제시한다** (서사 구조: 경험 → 원리 → 실무):
   ```
   ## [도입: 개인 경험 또는 비유]
   ## [핵심 질문: 왜 이게 중요한가]
   ## [원리: 보편적 개념 추출]
   ## [실무: 어떻게 적용할까]
   ## 정리 / keyword
   ```
5. 사용자 피드백을 기다린다.
   - 제목 선택 또는 수정 요청
   - 섹션 구조 변경 요청
   - 승인("ok", "좋아", "이대로") 시 Phase 2 진행

---

## Phase 2: 본문 작성

**실행 모드:** 단일 (메인 세션)

사용자가 승인한 구조 기반으로 섹션별 본문을 작성한다.

**Voice Profile 적용 규칙:**

1. **도입부**: 구체적 개인 경험 또는 일상 비유로 시작한다. 프로필에서 관찰된 패턴:
   - 아들 야구 연습 일화 — "연습 때 신경 쓰던 것을 시합에서는 잊으라" (#70)
   - 아들 운전 조수석 일화 — 위임과 신뢰의 비유 (#73)
   - 고등학교 수학 문제 에피소드 — 원리 이해 없이 답만 베끼면 다시 틀림 (#75)
   - 팀장/리더 첫 경험 고백 — "나 또한 처음 팀장이 되고 나서..." (#67)
   이 중 주제와 가장 연결되는 유형을 선택하거나 유사한 개인 경험 구조로 작성한다.

2. **문장 리듬**: 짧은 선언문 → 긴 설명문 교대. 단락당 3-5문장.
   핵심 원칙은 단독 행으로 세워 시각적으로 강조한다.

3. **어조**: "나"(경험 서술) + "우리"(독자 공감/개발자 집단) 교차.
   완충어("~인 것 같다", "~라고 생각한다", "적어도 내겐", "어쩌면", "아마도") 자연스럽게 사용.
   확신이 있는 주장에는 단언형("~이다", "~해야 한다") 사용.

4. **DO 패턴 적용** (profile의 특징적 표현):
   - "결국 ~라는 것이 내 결론이다" — 귀결형 마무리
   - "그렇다, ~" — 독자에게 선언하듯 시작
   - "(~했을까???)" — 괄호 안 반문·독백
   - "물론 ~ 만은 아니었다" — 반례 인정 후 주장 강화
   - "그런데", "하지만", "결국", "즉" — 자주 쓰는 연결어
   - "~는 게 맞다고 생각한다" — 조심스럽지만 명확한 주장

5. **AVOID 패턴 회피**: 다음 표현은 AI-tell로 금지.
   - "~을 통해서", "~에 있어서", "~을 기반으로 하여"
   - "다양한 측면에서", "결론적으로", "심층적으로 분석하면"
   - 이모지 사용 금지 (브런치 원문 전체에서 이모지 없음)
   - "~을 제고하다", "~를 도모하다" 등 공문서체

6. **강조 형식**: `>` 인용구로 핵심 강조, **굵은 글씨**로 개념 강조.
   소제목(##, ###)으로 논점을 3-6개로 명확히 구분.

7. **마무리**: "결국", "그러니" 같은 귀결 접속사로 마무리 신호.
   핵심 원칙을 한 문장으로 재요약하거나 행동 지침으로 맺는다.
   글 말미에 브런치 keyword 태그 섹션 포함.

**출력 형식:**
- 브런치 에디터 호환 마크다운
- 전체 분량: 800-1500자 (브런치 표준 칼럼 분량)
- 마지막에 권장 후속 작업 안내:
  ```
  > 💡 이 초안을 `/humanize-korean`에 넣으면 AI 티를 추가로 제거할 수 있습니다.
  ```
